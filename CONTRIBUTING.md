# Contributing

Thanks for considering contributing to RuleFlow PHP.

## Development Setup

```bash
composer install
composer test
composer lint
```

## Pull Request Guidelines

- Keep changes focused.
- Add tests for behavior changes.
- Update documentation when public APIs change.
- Explain the use case behind new operators or rule features.

## Design Boundary

RuleFlow should stay lightweight. Before adding platform-level features, open an issue and describe the production problem the feature solves.
